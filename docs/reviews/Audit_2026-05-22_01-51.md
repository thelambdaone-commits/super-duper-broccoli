# Refactoring Roadmap: Quant Agentic Trading Core

## Executive Summary
**Maintainability Score: 68/100**
*State of the Union: The codebase has evolved from a 'vibe-coded' prototype into a functional multi-agent system, but it suffers from severe 'God Object' congestion and leaky abstractions that will hinder scaling. While recent hardening has improved resilience, a structural decoupling of the orchestration and presentation layers is mandatory for production-grade reliability.*

## I. Architectural & Boundary Smells
| Location | Severity | Smell Detected | The Risk | The Fix |
| :--- | :--- | :--- | :--- | :--- |
| `telegram_scraper/telegram_listener.py` | **Critical** | God Object | 2,100+ lines mixing message parsing, command routing, state management, and media handling. Highly fragile. | Extract handlers into a `CommandRegistry`. Move core logic to `core/services`. |
| `core/orchestrator.py` | **High** | Boundary Violation | Directly manages task cleanup and queue monitoring while also enrichment logic. | Delegate enrichment to a `SignalEnricher` service. Use a dedicated `WorkerPool` manager. |
| `ledger/ledger_db.py` | **Medium** | Leaky Abstraction | Raw SQL strings scattered throughout methods. Difficult to migrate or test against mocks. | Implement a proper Repository Pattern or use an async ORM (SQLAlchemy/Tortoise). |

## IV. Code Hygiene & Sustainability
| Location | Severity | Smell Detected | The Risk | The Fix |
| :--- | :--- | :--- | :--- | :--- |

| `scripts/*.py` | **Medium** | Package Bloat | Some scripts import the entire `ServiceContainer` just to read a single secret. | Use a lightweight `SecretProvider` for standalone utility scripts. |

## Refactoring Priorities
1. **Decouple Telegram**: The `TelegramListener` is the single biggest point of failure. Extracting the command router was a good start; now move the handlers out.
2. **Schema Enforcement**: Replace all signal dictionaries with Pydantic models. This is the only way to ensure the AI 'brain' doesn't ingest hallucinations.
3. **Storage Abstraction**: Protect the Ledger from raw SQL. It makes the trading logic too coupled to SQLite's dialect.
