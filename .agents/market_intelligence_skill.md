# 📡 Market Intelligence Skill

*Moltbook-inspired Agentic Skill Document for prediction-market scanning, HMM volatility classification, and AI-powered market discovery.*

---

## 1. Purpose
Empowers the agent to dynamically monitor crypto-related binary prediction markets (Polymarket/Gamma), classify sentiment horizons (5m, 15m, 1h), and apply Hidden Markov Model (HMM) regime analysis to identify market context. Also provides AI-powered market discovery with scoring for optimal betting opportunities.

---

## 2. Trigger Conditions
* **Direct Commands**: User triggers `/btc5`, `/btc15`, `/btc1h`, `/eth5`, `/sol15`, `/xrp1h`, `/r` (regime report), or `/start`.
* **Market Discovery**: User triggers `/markets discover`, `/markets opportunities`, `/markets contrarian`
* **Autonomous Ingestion**: Intercepting incoming Telegram stream text for relevant tickers (BTC, ETH, SOL, XRP) or volume surges.
* **Hourly Cycle**: System maintenance tasks triggering automated opportunity scans.

---

## 3. Execution Steps

### Step A: Prediction Market Fetching
1. Query active Polymarket CLOB or Gamma markets using `PolymarketClient`.
2. Filter strictly for crypto-sentiment horizon markets matching the ticker and timeframe:
   * BTC 5m horizon slug pattern: `btc-updown-5m-*`
   * ETH 15m horizon slug pattern: `eth-updown-15m-*`
3. Retrieve current orderbook midpoints, volume, and last traded price.

### Step B: Market Discovery & Scoring (NEW)
1. Use `MarketDiscovery` class in [market_discovery.py](file:///home/ogj9f33gvvzc/quant-agentic-trading-core-v2/utils/market_discovery.py)
2. Score markets using 5-criteria algorithm:
   - **Liquidity Score** (30%): >= $100K = 100pts
   - **Volume Score** (25%): >= $1M = 100pts
   - **Edge Score** (25%): Optimal price range 0.25-0.75
   - **Competition Score** (15%): Lower odds = better value
   - **Sentiment Score** (5%): Tag-based sentiment analysis
3. Rank markets by total score and return top opportunities

### Step C: Volatility Regime Classification
1. Query the Hidden Markov Model (HMM) state engine in [regime_utils.py](file:///home/ogj9f33gvvzc/quant-agentic-trading-core-v2/utils/regime_utils.py).
2. Classify volatility context into `'LOW_VOLATILITY'`, `'HIGH_VOLATILITY'`, or `'TRENDING_BULLISH'/'TRENDING_BEARISH'`.

### Step C: Probability Calibration & Sizing
1. Extract outcome probabilities from orderbook prices (Yes price vs No price).
2. Run Platt Scaling/Isotonic regression calibration on historical outcomes to ensure probability calibration.
3. Compute initial Kelly Criterion size bounds in [portfolio_risk_engine.py](file:///home/ogj9f33gvvzc/quant-agentic-trading-core-v2/core/portfolio_risk_engine.py).

### Step D: Compiling Advisory Report
1. Format a premium, aesthetic terminal markdown message containing:
   * **Sentiment State**: `BULLISH 🟢`, `BEARISH 🔴`, or `NEUTRAL 🟡`.
   * **Interactive Bar Chart**: Visual character representations of consensus percentages.
   * **Market Statistics**: Volume, Base market horizons, and Slug.
   * **Regime Badge**: Active HMM Volatility Regime.
