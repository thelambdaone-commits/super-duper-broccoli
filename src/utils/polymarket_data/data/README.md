# Data Directory

This directory stores all fetched and processed data files.

## Structure

```
data/
├── dataset/              # Main dataset files
│   ├── orderfilled_*.parquet
│   ├── trades_*.parquet
│   ├── quant_*.parquet
│   └── users_*.parquet
├── latest_result/        # Latest 1000 records (CSV preview)
│   ├── orderfilled.csv
│   ├── trades.csv
│   ├── quant.csv
│   └── users.csv
├── data_clean/           # Cleaned data files
└── state.json            # Checkpoint state file
```

## File Types

### Parquet Files
- Compressed columnar format for efficient storage
- Generated by continuous fetching or batch processing
- Timestamped filenames: `<type>_YYYYMMDD_HHMMSS.parquet`

### CSV Preview Files
- Human-readable format
- Contains latest 1000 records
- Updated in real-time during continuous fetching

### State File
- JSON format tracking last processed block number
- Used for resume-from-checkpoint functionality

## Note

All files in this directory (except `.gitkeep` and this README) are ignored by git.
They should be backed up separately or uploaded to data repositories like HuggingFace.
